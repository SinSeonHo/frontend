### 일반적인컨트롤러 - 데이터 + html

### RestController - 데이터만

페이지를 주고받는 일반 컨트롤러와 다르게 데이터만 주고받는 방식 json형태로 주고받음

restcontroller에서는 매핑방식이 2개 더 추가됨

1)GET : Read 읽기/조회
2)POST : Create 생성
3)PUT/PATCH : Update 수정
4)DELETE : Delete 삭제

스프링에서 rest방식을 지원
@RestController
@DeleteMapping
@PutMapping
@PathVariable : 경로에 들어있는 값 변수에 담기
@RequestBody : json으로 넘어오는 데이터를 자바 객체에 연동시킴
@ResponseBody : 컨트롤러에서 리턴하는 값은 데이터임을 알림
ResponseEntity : data + 상태코드

CORS(Cross Origin Resource Sharing)
요청하는곳과 응답하는곳의 포트번호가 서로 틀릴경우 발생하는 에러
서버 : localhost:8080
클라이언트 : localhost:5500, localhost:5173 등 ~~

서버단에서 해결을 해야함 => CustomServletConfig.java 파일 서버쪽에 작성
